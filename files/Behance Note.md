# @@Behance Note

如何优化客户体验？如何提升运营效率？如何创造更多销售机会？

产品六个立足点：为主流用户创造有价值的，可用的，可行的产品体验。  +发展+实施

1. 需求立足点-需求模型：大众，主流，高频，刚需，>>范围圈定>>流量，转化率>>品牌认知>>砍掉不必要的路径
2. 体验立足点：可用性模型：：初次使用APP的体验不理想会丢失60%，口碑传播达到30%表示非常满意。
   1. 场景化（PSPS模型）（角色-场景-痛点-解决-功能）场景经常发生吗？问题经常发生吗？痛苦严重吗？暂时解决方案的风险有多少？
   2. 结构化（T架构设计）
   3. 流程化（单线程设计），不给客户思考的机会，如果有选择作为分支处理。（不应该问：你想吃湘菜还是川菜，应该问：走，去吃湘菜）
   4. 细致化（精细化设计）：细节决定成败，设计时越详细产品体验越好。（例如按键的点击区域，放大或下移）
   5. 品牌化（情感化设计）：产品的性格。（你的压力我知道）
3. 技术立足点：技术模型：弯道超车，未来可行的解决方案，并且可拆解出第一期迭代产品的立足点
4. 商业立足点：盈利模型：
   1. 变现：广告、电商、游戏、金融、服务
   2. 规模 不等于 效益 （1KM协同效益，小范围效益亏损，大范围效益亏损更大）
   3. 现金流 不等于 净利润 （先收加盟费，发展下线再返还，玩转正向现金流账期即可）
   4. 资本故事 不等于 我们的模式 （战略级的架构是梦想，短期的实施才是现实）
5. 环境的立足点：战略模式：
   1. 技术环境/：GPS定位精准
   2. 政策环境/：政府政策的变化
   3. 发展环境/：小米从MUI开始，到小米手机，智能硬件，
   4. 竞争环境/：不激烈慢慢孵化市场，，竞争激烈则大家一起哄市场。。
6. 团队立足点：执行模型：
   1. 动力和能力的XY轴：
   2. 创始人：leader
   3. 合伙人：PM RD UE
   4. 执行者：产品 运营 视觉 交互
7. 产品动机设计：
   1. 车辙设计：从痛点出发找需求为了任务设计任务 （需求）
   2. 糖线设计：从G点出发去钓鱼跳出任务设计任务  （套路）
8. 懂体验，懂运营，懂技术



@互联网+产品，首先了解传统行业的逻辑（行业、商务、产品），在需求层面迭代。最底层的需求：能挂到号，有更多医院。

线上场景：用户>>微信>>平台APP>>医院

线下场景：用户>>线下场景（网校，医院，公司）>>平台APP>>微信

如何监控渠道质量，跑脚本监控号源的真实性，每天一遍屏蔽无效链接。

用户体验扩展：是用户使用产品过程中建立起来的纯主观感受UX

1. 用户>>传统C端用户/合作方/医院和医生
2. 过程中>>使用线上产品的过程/线下的过程
3. 方便快捷/能赚到钱，能更好的赚钱/提升效率，解决矛盾
4. 用户主体，用户的五个特征



用户分析：

1. 用户规模和质量：活跃用户指标（周月），新增用户指标，用户构成指标（新老），用户留存率指标，每个用户总活跃天数指标 
2. （人均）启动次数指标，使用时长，访问页面，使用时间间隔
3. PV/UV::比例应该大于等于关键流程
4. 渠道分析：是否是真实的用户，（设备、时间、使用习惯）
5. 功能活跃/页面访问路径>>从用户细分到产品迭代>>漏斗分析
6. 用户属性分析：设备终端分析，网路及运营商分析，地域分析，用户画像
7. 每月签到：活动，每日签到：内容及功能



盈利模式：

1. 流量变现类>>广告/导流到付费业务/卖服务收取提成
2. 授权类>>接口费
3. 增值服务类>>会员特权/导用户信息给第三方
4. 传统开发类>>为医院开发更重软件/app/公众号::第三方付费



移动端的设计：聚焦于核心交互：屏幕尺寸、触屏操作、场景化、系统规范的应用、创新

1. 可以利用 X/Y/Z轴，用户持机方式
2. 合理删除、分层组织、适时隐藏、巧妙转移
3. 基础操作设计完成后，可增加高级快捷操作，如手势、语音等

桌面端的环境：

1. 个人工作重心
2. 安静的环境
3. 持续任务
4. 网络稳定

移动端的环境：

1. 临时性任务
2. 时刻移动，环境嘈杂
3. 随时被打断
4. 网络可能切换，中断

首先遵守规范，在进行创新。

#### 作品目的：通感设计

作品地址：http://www.uisdc.com/synaesthesia-design-language

评价：通感设计方法试图将产品质量与用户感知低损耗转移，增强体验的紧密的关联性与一致性。为数字产品创造有效设计语言的全新实用指南。本文章不是共享工具的介绍，也不是普通意义的通感设计手法，比如：Siri用波浪条纹设计让用户感受到声音。

通过对用户服务场景中全方位情感触点的设计，为用户传递一致的情感（传递的设计手段包括：视觉、听觉、触觉、信息引导、心理暗示等表达方式相互配合，达到一致心流频率）进而完成用户体验设计的相通，这将帮助团队输出始终如一的设计师理念。

#### 作品目的：CTA 设计方法

作品地址：http://www.uisdc.com/ux-design-call-to-action

评价：在绝大多数人的眼里，交互设计师的职责是对产品用户体验负责，毋庸置疑这肯定是正确的。但交互设计师并不是一个独立的个体，我们既服务于用户又服务于团队，服务于产品的商业价值，提升转化率也是交互设计中不得不考虑的重要一环。

福格模型简单来讲就是一个公式：B=MAT。B(behavior)代表行为，M(motivation)代表动机也就是用户需求，A(ability)代表用户使用的门槛，T(trigger)代表触发。也就是用户行为的产生需要用户需求为基础，需要保证产品的易用性，但是这还不够，在这个基础上我们还需要在产品中通过设计触发用户。

有三个环节不能忽略：注意力、传达、触发点。
